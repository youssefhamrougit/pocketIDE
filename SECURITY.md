# Security Policy

## Supported Versions

We currently support the latest stable release with security updates.

| Version | Supported          |
| ------- | ------------------ |
| latest  | ✅                 |
| < latest| ❌                 |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue in PocketIDE, **please do not open a public issue**.

Instead, report it privately by emailing **youssefmeddebhamrouni@gmail.com**.

Please include the following details in your report:

- **Type of vulnerability** (e.g., RCE, XSS, privilege escalation, data exposure)
- **Affected component** (editor, terminal, file system, Git, authentication, etc.)
- **Steps to reproduce** — be as specific as possible
- **Potential impact** — what an attacker could achieve
- **Suggested fix** (if you have one)

### What to Expect

1. **Acknowledgement** — We'll confirm receipt within 48 hours.
2. **Investigation** — We'll assess the severity and impact.
3. **Fix** — We'll develop and test a fix.
4. **Release** — We'll publish a patch and disclose the vulnerability responsibly.

We'll keep you informed at each step.

## Scope

The following are in scope:

- The PocketIDE Flutter application
- The backend API services
- Plugin/extension system
- Authentication and user data handling

The following are **out of scope**:

- Third-party plugins (report them to the plugin author)
- Issues in dependencies (report to the upstream maintainer)

## Recognition

We believe in recognizing security researchers who help keep our community safe. With your permission, we'll credit you in our security acknowledgements when we publish the fix.

Thank you for helping make PocketIDE secure for everyone.
